# 2. ERD (Current Real Design)

## 2.1 Database Landscape

Hệ thống hiện có nhiều lớp dữ liệu:
- **Mobile local DB (SQLite `pois.db`)**
- **Mobile in-memory localization store (không phải table)**
- **Mobile translation cache (SQLite)**
- **Backend DB (MongoDB)**
- **Legacy admin DB riêng (SQLite `pois-admin.db` trong `AdminWeb`)**

## 2.2 Mobile SQLite Tables

## Table: `pois` (mobile core POI)
- Mục đích: lưu 1 dòng lõi/POI cho vận hành offline (tọa độ + trigger metadata).
- Cột chính đang dùng:
  - `Id` (TEXT, PK): hiện thực tế gán bằng `Code`.
  - `Code` (TEXT): mã POI ổn định.
  - `Latitude` (REAL)
  - `Longitude` (REAL)
  - `Radius` (REAL)
  - `Priority` (INTEGER)
- Cột legacy được migrate thêm nhưng không đi theo model hiện tại:
  - `LanguageCode`, `Name`, `Summary`, `NarrationShort`, `NarrationLong`
- Index:
  - `IX_pois_Code`
  - `IX_pois_LanguageCode`

### Relationship
- Quan hệ logic 1-n với cache dịch qua `Code` (không có FK vật lý).

## Table: `poi_translation_cache`
- Mục đích: cache nội dung auto-translation theo ngôn ngữ đích.
- Cột:
  - `Key` (TEXT, PK): format `CODE|lang`
  - `Code` (TEXT)
  - `LanguageCode` (TEXT)
  - `Name` (TEXT)
  - `Summary` (TEXT)
  - `NarrationShort` (TEXT)
  - `NarrationLong` (TEXT)
  - `IsAutoGenerated` (BOOLEAN)
  - `CreatedAt` (DATETIME)

### Relationship
- Logic FK: `poi_translation_cache.Code` -> `pois.Code` (không enforce ở DB level).

## 2.3 Mobile Localization Store (Not Table Yet)

Hiện chưa có table `poi_localizations` trong SQLite.

Localization đang nằm trong:
- Dictionary in-memory `(Code, Lang) -> PoiLocalization`.
- Nguồn ban đầu từ `Resources/Raw/pois.json`.
- Có thể inject runtime từ sync backend hoặc translation service.

Điều này giải thích vì sao `Poi.Localization` là `[Ignore]` trong model SQLite.

## 2.4 Backend MongoDB Collections

## Collection: `pois`
- Trường:
  - `code` (String, unique)
  - `location` (GeoJSON Point, `coordinates` [lng, lat])
  - `radius` (Number)
  - `priority` (Number)
  - `languageCode` (String, default `vi`)
  - `name`, `summary`, `narrationShort`, `narrationLong`
  - `content` (legacy mixed field)
  - `isPremiumOnly` (Boolean)
  - `status` (`pending|approved|rejected` qua constant)
  - `submittedBy` (ObjectId -> `users`)
  - `rejectionReason`
  - `createdAt`, `updatedAt`
- Index:
  - `location` 2dsphere
  - `{code, status}`

## Collection: `poiRequests`
- Trường:
  - `poiData` (subdocument: code/location/radius/priority/language/content fields)
  - `status` (`pending|approved|rejected`)
  - `createdBy` (ObjectId -> `users`)
  - timestamps

## Collection: `adminpoiaudits`
- Trường:
  - `poiId` (ObjectId -> `pois`)
  - `adminId` (ObjectId -> `users`)
  - `action` (`APPROVE|REJECT`)
  - `previousStatus`, `newStatus`, `reason`
  - timestamps

## Collection: `users`
- Trường:
  - `email`, `fullName`, `password`
  - `role` (`ADMIN|USER|OWNER`)
  - `isPremium`, `isActive`, `qrScanCount`
  - timestamps

## 2.5 Web Admin DB (Legacy ASP.NET MVC)

## Table: `Pois` (`PoiRecord`)
- Trường:
  - `Id` (TEXT, PK)
  - `Code`
  - `LanguageCode`
  - `Latitude`, `Longitude`
  - `Radius`, `Priority`
  - `Name`, `Summary`, `NarrationShort`, `NarrationLong`
  - `UpdatedAt`
- Unique index:
  - `(Code, LanguageCode)`

Lưu ý: DB này độc lập backend Mongo, chỉ thuộc nhánh `AdminWeb`.

## 2.6 Why Some Columns Are Not Used Anymore

- Trên mobile table `pois`, các cột text/language được giữ lại chủ yếu do lịch sử migrate.
- Thiết kế hiện tại chuyển text khỏi `pois` để:
  - giảm coupling giữa geo core và nội dung ngôn ngữ,
  - hỗ trợ fallback/translation cache linh hoạt,
  - tránh nhân bản nhiều row/POI cho từng ngôn ngữ trong bảng lõi.

## 2.7 Why Localization Is Not Stored Directly In Main Table

- `pois` ưu tiên dữ liệu bất biến theo vị trí (geo core).
- Localization thay đổi theo:
  - ngôn ngữ yêu cầu,
  - fallback chain,
  - bản dịch runtime.
- Vì vậy localization được xử lý ở tầng service:
  - in-memory dictionary cho curated content,
  - translation cache table cho generated content.

## 2.8 Mobile DB vs Admin/Backend DB Differences

- **Mobile SQLite**: tối ưu runtime offline, chỉ chứa dữ liệu cần cho app chạy hiện trường.
- **Backend Mongo**: nguồn nghiệp vụ đầy đủ (status moderation, auth-related constraints, audit).
- **AdminWeb SQLite legacy**: nhánh cục bộ riêng, không tự động đồng bộ 2 chiều với Mongo.

## 2.9 Source Of Truth (Current Reality)

- **Server-side nghiệp vụ POI duyệt**: Mongo `pois` (status `APPROVED`) là nguồn cấp cho API mobile sync.
- **Mobile runtime localization curated**: `Resources/Raw/pois.json` + `LocalizationService` in-memory.
- **Auto-translation runtime**: `poi_translation_cache` + inject vào in-memory localization.
- **Rủi ro Source-of-Truth kép**: nhánh `AdminWeb` SQLite có thể lệch so với Mongo nếu vận hành song song.
